# 小红书 AI 热点图文笔记（xhs-hotspot-note）

把"最近热点"加工成一篇**小红书风格的图文笔记**。核心不是单纯转述新闻，而是用**"AI 实用向"角度切入**——落点在"这件事用 AI 怎么分析 / 能做个什么 / 对你有什么用"，这样既蹭到热度，又有实用钩子和差异化，而不是和满屏搬运号撞车。

适用平台：先专注小红书图文。抖音适配（短视频脚本）暂不在本 skill 范围内。

## 为什么这样设计

小红书的推荐逻辑奖励"有用 + 有情绪共鸣 + 能收藏"的内容，纯新闻复述容易同质化、留不住人。把热点 + AI 实用价值绑定，能同时拿到热点流量和长尾收藏，也发挥了"作者本来就在用 AI"的优势。

## 重要前提：发布环节的现实约束

小红书没有面向个人的官方笔记发布 API，所以本 skill **不调用任何第三方/逆向接口**（那类方案有封号风险）。发布走两种安全路径：

1. **内容准备（默认）**：生成可直接复制的笔记文件，用户手动粘贴发布。完全合规、零风险。
2. **浏览器半自动（可选）**：在用户已登录的创作者后台网页上，用 Claude in Chrome 插件帮忙填入标题/正文/标签。**最后的"发布"按钮务必由用户本人点击**，降低账号风险。

## 工作流程

### 第 1 步：确定热点

询问或确认用户想蹭哪个热点。如果用户没指定，就用联网搜索抓取候选：

- 科技/AI 媒体热点（优先，和 AI 实用向最契合）
- 全网泛热搜（微博热搜、百度热搜等）作为补充

从候选里**挑能切出 AI 实用角度的**（能延伸到"用 AI 做点什么"的话题最佳），给用户 3–5 个选项让其选一个，或按相关度自动选一个并说明理由。务必基于真实检索结果，不要编造热点。

### 第 2 步：定 AI 实用角度

为选定热点找一个具体的"实用钩子"，例如：
- 用 AI 工具对这个热点做个快速分析 / 对比 / 整理
- 借这个热点教一个能复用的 AI 用法
- 把热点信息加工成读者能直接拿走用的清单/模板

角度要具体、可落地，避免空泛的"AI 很厉害"。

### 第 3 步：生成笔记

按 `templates/note-structure.md` 的结构产出一篇完整笔记，包含：

- **标题**：参考 `templates/title-templates.md`，带 1–2 个 emoji，有数字/对比/痛点钩子，控制在 20 字内
- **正文**：开头 3 秒抓人 → 干货分点 → 结尾引导互动（点赞收藏/评论提问），分段清晰、多用 emoji 分隔、口语化
- **话题标签**：参考 `templates/tags.md`，5–8 个，热点词 + 垂直词 + 流量词混搭
- **配图建议**：封面文案 + 内页配图思路（几张、各放什么），小红书竖图建议 3:4

### 第 4 步：输出文件

把笔记保存成一个 Markdown 文件（标题、正文、标签、配图建议分区清晰），方便用户复制。文件命名建议含日期和主题，例如 `小红书-20260629-热点主题.md`。

参考 `examples/` 里的示范笔记把控质量。

### 第 5 步（可选）：浏览器半自动发布

仅当用户明确要求"帮我填到后台/发布"时执行：

1. 确认 Claude in Chrome 插件已连接、用户已登录 creator.xiaohongshu.com
2. 用插件导航到发布页，把封面/图片提示用户上传（或按用户提供的图）
3. 把标题、正文、标签填入对应输入框
4. **停在发布前**，请用户本人检查并点击"发布"

## 质量自查

产出前快速核对：标题有没有钩子？开头能不能 3 秒抓住人？干货是不是真有用、能收藏？有没有引导互动？标签够不够、相关不相关？热点信息是否真实（来自检索而非编造）？
